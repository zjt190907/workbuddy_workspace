# 工作区的命名与架构规范

---

## 一、顶层文件命名

| 序号 | 文件名 | 用途 |
|------|--------|------|
| 00 | `00_全局控制台.md` | 工作手册、新项目入口 |
| 01 | `01_全局复利与踩坑日志.md` | 错误与好经验记录 |
| 02 | `02_工作区的命名与架构规范.md` | 本文件 |
| 03 | `03_项目创建sop.md` | 项目创建标准流程 |
| - | `AGENTS.md` | Agent 行为指引 |

- 序号前缀表示优先级/读取顺序
- 全局文件只在 `D:\workBuddy_workspace\` 顶层存在一份，禁止复制到项目内

---

## 二、项目目录命名

### 格式

```
PROJECT_序号_项目名
```

### 规则

- **序号**：三位数字，从 001 开始递增（`PROJECT_001`、`PROJECT_002`...）
- **项目名**：小写英文 + 下划线，简洁表意（如 `stock_tracker`、`blog_system`）
- **位置**：统一放在 `D:\workBuddy_workspace\` 下

### 示例

```
PROJECT_001_stock_tracker/
PROJECT_002_blog_system/
PROJECT_003_api_gateway/
```

---

## 三、项目内部结构

项目内只放项目自己的文档和代码，**不放全局控制台全文**。

```
PROJECT_001_stock_tracker/
├── README.md              ← 项目说明
├── docs/                  ← 项目文档
├── src/                   ← 源代码
├── tests/                 ← 测试
└── ...
```

具体结构根据项目类型（前端/后端/全栈/数据分析等）灵活调整。

---

## 四、禁止事项

- 禁止在顶层以外的地方创建项目
- 禁止在项目内复制全局文件（00~03、AGENTS.md）
- 禁止跨项目混放文件，每个项目独立目录
