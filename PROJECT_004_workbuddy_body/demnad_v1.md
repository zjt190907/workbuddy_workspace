桌面Agent外接显示屏 — 交互与显示需求规格说明
1. 项目背景
硬件：ESP32-C3 SuperMini + ST7789 1.54寸 240×240 TFT屏
当前状态：基础显示程序已烧录，但效果单调、交互性弱，无法随Agent工作状态动态变化。
目标：让显示屏成为Agent的“面部”与“思维窗口”，拥有丰富的表情动画、状态感知、思考内容实时显示，大幅提升交互体验与观赏性。

2. 核心交互逻辑
Agent运行在PC端，通过串口（UART）或Wi-Fi（TCP/WebSocket）与ESP32屏端实时通信。
Agent主动向屏幕推送状态指令和内容数据，屏幕端负责解析并渲染对应的动画与界面。

必须支持的状态（可扩展）：

状态	说明
待机（Idle）	Agent无任务，等待用户指令
思考（Thinking）	接收任务后、开始处理前的推理阶段
工作中（Working）	执行工具调用、联网搜索、代码运行等
任务完成（Completed）	成功返回结果，准备展示或总结
休眠（Sleep）	长时间无交互或用户主动休眠
错误/警告（Error）	遇到异常或失败（可选，后期加入）
3. 详细显示需求
3.1 待机状态
表情：微微眨眼、偶尔头部轻微晃动或左右张望，表现“在线且放松”。

动画至少包含：缓慢眨眼（2~3秒一次）、眼神随机微小转动。

屏幕可显示一句随机“待机语”（如“我在哦～”），字号小而靠下。

若检测到用户靠近或Agent被唤醒，需有唤醒过渡动画（表情变亮、眼睛睁大）。

3.2 思考状态
表情：眼睛向上看或斜视，可加入挠头、头上冒问号、虚线圆点等动画元素。

核心功能：显示思考内容

Agent应将当前的思考步骤/内部独白以短文本形式发送到屏幕。

显示方式：滚动字幕或打字机效果，文字在表情下方或对话框气泡中出现。

示例文本：“嗯…让我想想”“正在搜索相关文档…”“分析用户意图中…”

文字需要自动换行，240×240空间建议底部40像素高度用于文字滚动。

思考状态持续期间，表情应保持动态（眼珠可随“思考”节奏转动）。

3.3 工作中状态
表情：专注、眼睛微微眯起或戴上“护目镜”特效（可选），旁边可显示迷你进度条或齿轮旋转动画。

可根据工作类型切换表情变体：

搜索网络：眼睛变成放大镜图标或雷达动画。

执行代码：显示小齿轮或 >_ 符号动态。

读取文件：眼镜或书本动画。

同样支持显示简短的工作内容提示，如“正在访问API…”，同思考内容的文字显示区域复用。

3.4 任务完成状态
表情：开心、眼睛弯成月牙，可以有烟花、星星、撒花粒子特效（简单实现即可）。

显示一行结果摘要，例如“找到了42条结果”或“代码已生成”，停留2~3秒后自动切换回待机。

完成动画需有高潮感：表情跳跃一下或放大一下后恢复。

3.5 休眠状态
屏幕显示闭眼或睡颜（ZZZ），亮度可调低（如果硬件支持背光控制）。

可缓慢呼吸灯效（表情明暗变化）。

从休眠唤醒时有打哈欠伸懒腰动画，流畅过渡到待机。

3.6 表情多样化
以上每种状态至少准备3种不同的表情变体，Agent可随机选择或根据上下文切换，避免重复单调。

表情应基于向量/图层绘制（利于形变和动画），而非纯图片切换。建议使用精灵图或图形绘制API（TFT_eSPI可直接画圆、弧、多边形来构造面部）。

所有状态切换都需要0.3~0.5秒的平滑过渡动画（如淡入淡出、变形插值），不能生硬跳变。

4. 通信协议要求
为解耦PC端Agent与ESP32显示逻辑，定义轻量指令格式。

物理层： 优先使用USB模拟串口（稳定低延迟），亦可扩展Wi-Fi TCP。
数据格式： JSON或简单字符串协议，每条指令以换行分隔。
示例指令集（需实现）：

json
{"cmd":"state","value":"thinking"}
{"cmd":"think_text","text":"正在分析您的提问..."}
{"cmd":"state","value":"working","subtype":"code"}
{"cmd":"state","value":"completed","summary":"任务成功！"}
{"cmd":"state","value":"idle"}
{"cmd":"sleep"}
{"cmd":"wake"}
屏幕端需返回简单的ACK或状态（可选）。

5. UI/UX 设计指导
分辨率240×240，主体表情占据中央约160×160区域，留出边缘用于对话框或状态图标。

文字使用抗锯齿小字体，确保清晰。思考内容最多显示两行，自动滚动。

色彩鲜艳、圆润可爱风格，主色调建议暖色或根据Agent性格设定。

背景可纯色或微渐变，不要抢戏。

6. 性能与稳定性要求
屏幕刷新率保证在15fps以上，动画流畅不撕裂。

状态指令响应延迟<100ms，即Agent发出指令后屏幕需在0.1秒内开始动画过渡。

通信中断时（USB拔出），屏幕应自动切换到“失联”表情（例如雪花屏或失落表情）并尝试重连。

代码结构清晰，预留添加新状态和新表情的接口。

7. 待扩展能力（V1.1 可考虑）
触摸反馈（如果后续更换触摸屏，可支持摸头杀反应）。

语音唤醒同步动效。

时间/天气轻展示（从Agent获取数据）。

动态切换主题/配饰（帽子、眼镜等）。
