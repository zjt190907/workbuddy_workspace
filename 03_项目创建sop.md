# 项目创建 SOP

---

## 创建流程

### 第一步：扫描全局文档

按以下顺序读取顶层文件：

1. `00_全局控制台.md` — 确认工作手册和最高指令
2. `01_全局复利与踩坑日志.md` — 避坑 + 复用已有经验
3. `02_工作区的命名与架构规范.md` — 确认命名与架构规范
4. `03_项目创建sop.md` — 确认创建流程（本文件）

### 第二步：确定项目信息

- 项目名称（英文，下划线分隔）
- 项目类型（前端/后端/全栈/数据分析/工具等）
- 项目序号（查看现有项目，取最大序号 +1）

### 第三步：在顶层创建项目目录

```
D:\workBuddy_workspace\PROJECT_序号_项目名\
```

### 第四步：初始化项目

根据项目类型初始化：

- 创建 `README.md`（项目说明）
- 创建项目所需的目录结构（src/、docs/、tests/ 等）
- 初始化版本控制（如需要）
- 初始化依赖管理（package.json / pom.xml / requirements.txt 等）

### 第五步：记录

- 在 `01_全局复利与踩坑日志.md` 中记录创建过程中遇到的问题或好经验
- 项目开发在项目目录内进行，避免上下文污染

---

## 注意事项

- 全局文档不复制到项目内，项目通过引用顶层文件获取规范
- 项目序号严格递增，不跳号不复用
- 创建完成后告知简项目路径和基本结构
